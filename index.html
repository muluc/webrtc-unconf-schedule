Scalable Video Coding in VP8, VP9 and AV1

Juliusz Chroboczek
During a videoconference, the receivers have different links, and therefore need to receive video at different throughputs. The traditional solution to this problem is called "simulcast", where the sender encodes multiple independent streams at different qualities and the server selects which stream is forwarded to each receiver. The successor to simulcast is called Scalable Video Coding (SVC). With SVC, the sender sends a single "scalable" stream, which interleaves low- and high-quality frames. The server decides, for each receiver, which frames to forward and which packets to strip. The main advantage of SVC over simulcast is that switching drop strategies is a lightweight operation, and that SVC can therefore react to fine-grained congestion indications. The Galene video server (https://galene.org) recently acquired the ability to perform SVC stripping. In this talk, I'll describe how SVC works in VP8, VP9 and AV1, how it is implemented in Galene, and what new features we can look forward to with the upcoming AV1 codec.
